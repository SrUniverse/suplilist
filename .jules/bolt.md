## 2024-05-30 - Prevent Deep State Cloning in Redux-like State Managers
**Learning:** In `src/state/state-manager.js`, the global state is deeply frozen. The `setState` method was previously invoking `this.export()` (which performs `JSON.parse(JSON.stringify(this._state))`) before passing it to the path-updating function `_setPath`. This caused O(N) full state tree duplication on every slice update, creating severe bottleneck for rapid successive mutations. Since `_setPath` naturally creates shallow clones using the spread operator (`...obj`) along the updated branch, the deep copy was entirely redundant.
**Action:** When updating a deep nested, immutable state tree, traverse and update via shallow structural sharing (path cloning) instead of deep-cloning the entire global state upfront. Use `this._state` directly instead of a full exported snapshot.
