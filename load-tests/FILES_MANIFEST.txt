=== SETUP INSTRUCTIONS ===

1. INSTALLATION (30 minutes)
   - Read: load-tests/INSTALLATION.md
   - Install K6: brew install k6 (or platform-specific)
   - Install Locust: pip install locust
   - Verify: k6 version, locust --version

2. CONFIGURATION (5 minutes)
   - Create .env file with BASE_URL and other settings
   - Configure SLA targets in config/sla-targets.yaml
   - Set notification webhooks (Slack/PagerDuty)

3. FIRST TEST (20 minutes)
   - Start application server: npm run dev:server
   - Read: load-tests/TEST_PROCEDURES.md
   - Run normal load test: TEST_SCENARIO=normal ./load-tests/scripts/run-load-tests.sh
   - Check results: ./load-tests/scripts/performance-validation.sh results/k6-*.json

4. BASELINE (5 minutes)
   - Save baseline: SAVE_BASELINE=true TEST_SCENARIO=normal ./load-tests/scripts/run-load-tests.sh
   - Baseline saved to: load-tests/baselines/normal-baseline.json

5. CI/CD SETUP (10 minutes)
   - GitHub Actions workflow is ready at: .github/workflows/load-testing.yml
   - Configure secrets: SLACK_WEBHOOK_URL, PAGERDUTY_KEY
   - Tests will run automatically on schedule

=== QUICK COMMANDS ===

# Run normal load test
TEST_SCENARIO=normal ./load-tests/scripts/run-load-tests.sh

# Run peak load test
TEST_SCENARIO=peak ./load-tests/scripts/run-load-tests.sh

# Run stress test
TEST_SCENARIO=stress ./load-tests/scripts/run-load-tests.sh

# Validate performance
./load-tests/scripts/performance-validation.sh results/k6-*.json normal

# Start Locust Web UI
cd load-tests/locust && locust -f locustfile.py --host http://localhost:3000

# Run with K6 directly
k6 run load-tests/k6-tests/normal-load.js --vus 100 --duration 12m

# Check help
./load-tests/scripts/run-load-tests.sh --help
